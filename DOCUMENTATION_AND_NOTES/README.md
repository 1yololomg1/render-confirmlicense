# Documentation and Notes

This folder contains all documentation, notes, instructions, and non-build files for the CONFIRM project.

## Contents

### Documentation Folders
- **03_DOCUMENTATION/** - User manuals, technical specifications, and quick start guides
- **02_SERVER/** - Server-side code, deployment guides, and server documentation

### Markdown Documentation Files
- **README.md** - Main project readme
- **APP_STRUCTURE.md** - Application structure documentation
- **ALL_FIXES_SUMMARY.md** - Summary of all fixes applied
- **BUILD_VERIFICATION_REPORT.md** - Build verification documentation
- **CHANGES_MADE.md** - Record of changes made to the project
- **FINAL_BUILD_AND_TEST.md** - Final build and testing documentation
- **PRE_BUILD_CHECKLIST.md** - Checklist before building
- **PRE_RELEASE_CHECKLIST.md** - Checklist before release
- **README_RELEASE_STATUS.md** - Release status information
- **SAFE_CHART_FIX.md** - Documentation about chart fixes
- **BETA_ISSUES_TO_FIX.md** - List of beta issues to fix

### Text Files
- **BUILD_INSTRUCTIONS.txt** - Detailed build instructions
- **COPYRIGHT_NOTICE.txt** - Copyright information
- **confirmpython3.txt** - Python 3 compatibility notes
- **confirm python3.txt** - Additional Python 3 notes

### Configuration and Admin Files
- **firebase-database-rules.json** - Firebase database security rules
- **license_admin.html** - License administration interface
- **license_admin_secure.html** - Secure license administration interface
- **server.mjs** - Server configuration file (root level)

## Purpose

This folder is organized to keep all documentation, notes, and non-essential build files separate from the actual build files needed to create the executable. This makes it easier to:

- Find build files quickly
- Keep documentation organized
- Share only build files when needed
- Maintain clear separation between code and documentation
